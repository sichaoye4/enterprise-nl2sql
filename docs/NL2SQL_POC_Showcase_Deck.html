Lessons for higher accuracy

What makes an NL2SQL system accurate enough to trust?

High accuracy is not a property of the LLM alone. It is the outcome of disciplined meaning resolution, high-quality context, layered verification and a deliberate learning loop.

01

Resolve meaning before retrieving schema

Business language is overloaded. Define governed metrics, dimensions, entities, time semantics and join paths so the system can distinguish concepts such as GMV, paid GMV and net revenue before it writes SQL.

02

Treat context as a curated data product

More schema is not better context. Favor certified, eligible data assets with clear grain, ownership, descriptions, sample values and caveats. Retrieve the smallest relevant set that supports the question.

03

Verify semantics, not only syntax

Executable SQL can still be wrong. Combine static checks with semantic validation of metric, aggregation, time, joins and grain; then use preview execution and an independent quality review for uncertain cases.

04

Make learning part of the product

Capture questions, route decisions, context, SQL, outcomes and user corrections. Convert repeatable failures into regression cases, stronger semantic definitions, retrieval assets and tested guardrails.

Accuracy operating loop

Reduce uncertainty early, verify before impact, and make every confirmed outcome reusable.
